# Подробное руководство по расширенному редактору XR Shop

Внутриигровой редактор позволяет управлять серверным `shop.json`, не открывая и не передавая сам JSON-файл клиенту. Сервер создаёт временный типизированный черновик, проверяет каждое изменение и только после отдельной команды сохранения атомарно заменяет рабочую конфигурацию.

Редактор открывается командой:

```text
/ms admin editor_extension
```

Требуемое право:

```text
minecraftshop.admin.editor
```

По умолчанию административное право доступно операторам с уровнем не ниже 2. При установленном Forge-совместимом обработчике Permission API право можно назначить отдельно.

## 1. Как устроена сессия редактора

При открытии редактора сервер:

1. Проверяет право `minecraftshop.admin.editor`.
2. Закрывает предыдущую редакторскую сессию этого администратора, если она существовала.
3. Создаёт случайный `editor_session_id`.
4. Копирует текущий проверенный `ShopSnapshot` в серверный черновик.
5. Запоминает текущую revision как `base_revision`.
6. Передаёт авторизованному клиенту типизированный `EditorDto`.

В `EditorDto` временно присутствуют `purchase_commands`, необходимые для редактирования товаров. Они хранятся только в памяти клиента и удаляются после закрытия экрана, отмены, сохранения или отключения от сервера.

Черновик остаётся на сервере. Клиент не получает `shop.json` как файл и не отправляет серверу готовый JSON магазина.

Стандартный тайм-аут редактора — 900 секунд. Он задаётся параметром `editor_timeout_seconds` в `server.json`. Каждая корректно принятая операция продлевает срок жизни сессии.

## 2. Области экрана

Экран редактора состоит из трёх логических областей.

### Панель операций

Слева находятся пять полей ввода и кнопки операций:

1. `category_id` — текущая или исходная категория.
2. `slot_id` — текущий или исходный товар.
3. `новый/целевой ID` — новый ID копии либо категория назначения при переносе.
4. `поле` — имя редактируемого поля или индекс команды.
5. `значение JSON или команда` — новое значение поля либо строка команды.

### Список категорий и товаров

В центральной части отображается серверный черновик:

- верхний уровень — категории;
- вложенные строки — товары выбранной категории;
- выключенные товары отображаются приглушённым цветом.

Щелчок по категории заполняет `category_id` и очищает `slot_id`. Щелчок по товару заполняет оба идентификатора. Список прокручивается колесом мыши.

### Предварительный просмотр

Если ширины экрана достаточно, справа отображается упрощённый предварительный просмотр выбранной категории и её карточек.

Предпросмотр предназначен для быстрой проверки названий, цен, размеров и основных цветов. Окончательный интерфейс игрока следует проверять в обычном магазине через `/ms open`, поскольку реальный `StoreScreen` дополнительно учитывает разрешение, GUI Scale, права конкретного игрока, страницы, прокрутку, иконки, tooltip и наследование стилей.

## 3. Синтаксис вводимых значений

Операции изменения полей принимают одно JSON-значение, а не произвольный текст. Тип значения должен соответствовать типу поля.

### Строка

Строки обязательно заключаются в двойные кавычки:

```json
"Новое название"
```

Примеры:

```json
"Ресурсы"
"minecraft:diamond"
"#FF55AAFF"
"minecraftshop.vip"
```

### Целое число

```json
250
```

Суффиксы Java, пробелы-разделители и кавычки не используются:

```json
100
86400
0
```

### Дробное число

Дробные значения нужны только для громкости и высоты звука:

```json
0.7
1.25
```

### Логическое значение

```json
true
```

или:

```json
false
```

### Наследуемое подтверждение покупки

Поле товара `confirmation_enabled` допускает `null`. Это означает наследование глобального значения `ui.confirmation_enabled`:

```json
null
```

### Массив строк

Описание категории или товара задаётся массивом:

```json
["Первая строка", "Вторая строка", "Цена: {price} XR"]
```

### Строка с кавычками

Кавычки внутри JSON-строки экранируются обратной косой чертой. Например, display NBT:

```json
"{display:{Name:'{\"text\":\"Особый предмет\"}'}}"
```

## 4. Идентификаторы

Редактор использует два ID:

- `category_id` — глобально уникальный ID категории;
- `slot_id` — уникальный ID товара внутри своей категории.

Формат обоих ID:

```text
[a-z0-9_.-]+
```

Примеры корректных ID:

```text
resources
vip-items
weapons.tier_2
diamonds_16
```

Некорректные ID:

```text
Ресурсы
VIP Items
minecraft:diamond
item/one
```

Длина ID ограничена `limits.max_id_length` из `server.json`, по умолчанию — 64 символа.

`slot_id` не является индексом массива, номером GUI-слота или экранной координатой. Пара `(category_id, slot_id)` однозначно определяет товар.

## 5. Операции над категориями

### Создание категории

1. Введите новый ID в `category_id`.
2. Нажмите `+ кат.`.

Сервер создаст категорию со следующими начальными значениями:

- `category_id` равен введённому ID;
- `title` временно равен ID;
- `enabled` равно `true`;
- `order` устанавливается после существующих категорий;
- список товаров пуст;
- иконка использует безопасное стандартное значение модели `minecraft:barrier`.

После создания рекомендуется сразу изменить `title`, `description`, `icon` и оформление.

### Удаление категории

1. Выберите категорию или введите её ID.
2. Нажмите `− кат.`.
3. В течение трёх секунд нажмите кнопку повторно.

Удаление категории удаляет из черновика и все находящиеся в ней товары. До сохранения рабочий магазин не меняется.

### Копирование категории

1. В `category_id` укажите исходную категорию.
2. В `новый/целевой ID` укажите новый `category_id`.
3. Нажмите `Коп. кат.`.

Копируются все поля категории, стили и товары. Новый ID должен быть глобально уникальным. К названию копии добавляется `(копия)`.

### Изменение порядка категории

Отдельная кнопка перемещения не требуется. Измените поле `order`:

```text
category_id: resources
поле: order
значение: 0
```

Затем нажмите `Поле кат.`. Чем меньше `order`, тем раньше категория отображается. При одинаковом `order` используется технический ID.

## 6. Поля категории

Для изменения категории заполните `category_id`, имя поля и JSON-значение, затем нажмите `Поле кат.`.

| Поле | Тип | Назначение |
|---|---:|---|
| `category_id` | string | Уникальный технический ID. Изменение проверяется на формат и конфликт. |
| `title` | string | Отображаемое название, может быть на русском. |
| `description` | string[] | Многострочная подсказка категории. |
| `enabled` | boolean | Доступность категории. Выключенная категория не входит в обычный DTO. |
| `order` | int | Порядок сортировки. |
| `permission` | string | Дополнительное Forge-право или пустая строка. |
| `tab_width` | int | Индивидуальная ширина вкладки; `0` использует глобальную ширину. |
| `padding` | int | Индивидуальный внутренний отступ. |
| `product_icon_size` | int | Общий размер иконок товаров категории от 8 до 64; применяется, когда у товара `icon.size` равен `0`. |

Вложенные поля иконки:

| Поле | Тип | Назначение |
|---|---:|---|
| `icon.type` | string | `"ITEM"` или `"TEXTURE"`. |
| `icon.item` | string | Registry ID предмета для `ITEM`. |
| `icon.count` | int | Отображаемое количество от 1 до 99. |
| `icon.size` | int | Для товара: `0` наследует `product_icon_size` категории, 8–64 задаёт собственный размер. |
| `icon.display_nbt` | string | Ограниченный display NBT. |
| `icon.texture` | string | Клиентский `ResourceLocation` для `TEXTURE`. |

Вложенные поля `style` перечислены в разделе «Оформление».

Пример переименования:

```text
category_id: resources
поле: title
значение: "Ресурсы"
кнопка: Поле кат.
```

Пример права:

```text
category_id: vip
поле: permission
значение: "minecraftshop.category.vip"
```

Пустое право задаётся так:

```json
""
```

## 7. Операции над товарами

### Создание товара

1. Укажите существующую категорию в `category_id`.
2. Укажите новый ID товара в `slot_id`.
3. Нажмите `+ товар`.

Новый товар намеренно создаётся с `enabled: false`, `icon.size: 0` и пустым `purchase_commands`. Благодаря этому незавершённый товар нельзя купить, а его размер сразу наследуется из категории.

Рекомендуемый порядок настройки нового товара:

1. Изменить `title` и `description`.
2. Установить `price_xr`.
3. Настроить иконку.
4. Добавить хотя бы одну исполняемую команду.
5. Настроить лимит, cooldown, сообщения и стиль.
6. Выполнить `Проверить`.
7. Только после этого установить `enabled` в `true`.

### Удаление товара

1. Выберите товар.
2. Нажмите `− товар`.
3. Повторно нажмите кнопку в течение трёх секунд.

### Копирование товара

1. Укажите исходные `category_id` и `slot_id`.
2. В `новый/целевой ID` укажите новый `slot_id`.
3. Нажмите `Коп. тов.`.

Копия создаётся в той же категории. Новый `slot_id` должен быть свободен. К названию добавляется `(копия)`, а `enabled` принудительно устанавливается в `false`, чтобы копию нельзя было купить до проверки.

### Перенос товара

1. В `category_id` укажите исходную категорию.
2. В `slot_id` укажите переносимый товар.
3. В `новый/целевой ID` укажите категорию назначения.
4. Нажмите `Перенести`.

`slot_id` при переносе не меняется. Если в целевой категории уже существует такой `slot_id`, операция будет отклонена.

## 8. Поля товара

Для изменения товара заполните `category_id`, `slot_id`, имя поля и JSON-значение, затем нажмите `Поле тов.`.

| Поле | Тип | Назначение |
|---|---:|---|
| `slot_id` | string | Технический ID внутри категории. |
| `title` | string | Отображаемое название. |
| `description` | string[] | Многострочное описание и tooltip. |
| `enabled` | boolean | Доступность товара. |
| `price_xr` | long | Цена в целых XR, не может быть отрицательной. |
| `order` | int | Порядок товара среди остальных. |
| `page` | int | Номер страницы, начиная с 0. |
| `row` | int | Предпочтительная строка сетки, начиная с 0. |
| `column` | int | Предпочтительный столбец сетки, начиная с 0. |
| `permission` | string | Дополнительное право или пустая строка. |
| `confirmation_enabled` | boolean/null | Индивидуальное подтверждение; `null` наследует UI. |
| `purchase_button_text` | string | Текст кнопки покупки. |
| `purchase_limit` | long | Максимум покупок за период; `0` отключает лимит. |
| `limit_period_seconds` | long | Период лимита в секундах. |
| `cooldown_seconds` | long | Задержка между покупками в секундах. |
| `success_message` | string | Сообщение после полного успеха. |
| `error_message` | string | Дополнительное сообщение при ошибке покупки. |

Иконка товара использует поля `icon.type`, `icon.item`, `icon.count`, `icon.size`, `icon.display_nbt`, `icon.texture`. Только размер поддерживает наследование: `icon.size: 0` берёт `product_icon_size` текущей категории. После переноса такой товар автоматически получает размер целевой категории; остальные свойства иконки не меняются.

Звук товара:

| Поле | Тип | Назначение |
|---|---:|---|
| `sound.sound` | string | Registry ID звукового события. |
| `sound.volume` | float | Громкость от 0 до 4. |
| `sound.pitch` | float | Высота от 0.1 до 4. |

Пример цены:

```text
category_id: resources
slot_id: diamonds_16
поле: price_xr
значение: 100
```

Пример описания:

```text
поле: description
значение: ["Выдаёт 16 алмазов", "Цена: {price} XR"]
```

Даже описание из одной строки вводится как массив: `["Выдаёт один алмаз"]`. Значение `"Выдаёт один алмаз"` без квадратных скобок будет отклонено с ошибкой `Expected BEGIN_ARRAY but was STRING`.

Пример лимита «5 покупок в сутки»:

```text
поле: purchase_limit
значение: 5
```

затем:

```text
поле: limit_period_seconds
значение: 86400
```

Пример cooldown в 30 секунд:

```text
поле: cooldown_seconds
значение: 30
```

## 9. Исполняемые команды товара

Команды являются единственным способом выдать результат покупки. Иконка товара не участвует в выдаче.

Команды:

- сохраняются без начального `/`;
- выполняются серверной консолью;
- выполняются последовательно в порядке списка;
- не отправляются обычным покупателям;
- проверяются при добавлении, сохранении и непосредственно перед списанием XR.

Поддерживаемые заполнители:

| Заполнитель | Серверное значение |
|---|---|
| `${player}` | Точное имя игрока. |
| `${uuid}` | UUID игрока. |
| `${price}` | Актуальная серверная цена. |
| `${category_id}` | Актуальный ID категории. |
| `${slot_id}` | Актуальный ID товара. |
| `${transaction_id}` | Уникальный ID транзакции. |

Другие заполнители `${...}` запрещены. Старый формат `{player}` не поддерживается. JSON и NBT используют обычные фигурные скобки и не разбираются как placeholders.

### Добавление команды

1. Выберите товар.
2. В поле значения введите команду как обычный текст, без JSON-кавычек и начального `/`.
3. Нажмите `+ команда`.

Пример:

```text
give ${player} minecraft:diamond 16
```

### Изменение команды

1. В поле `поле` укажите индекс команды, начиная с 0.
2. В поле значения введите новую команду.
3. Нажмите `Команда`.

Например, изменение первой команды:

```text
поле: 0
значение: give ${player} minecraft:diamond 32
```

### Удаление команды

1. В поле `поле` укажите индекс команды.
2. Нажмите `− команда` два раза в течение трёх секунд.

Если после удаления команд включённый товар останется без `purchase_commands`, операция будет отклонена. Сначала выключите товар или добавьте заменяющую команду.

### Последовательность команд

Для товара можно добавить несколько команд:

```text
give ${player} minecraft:diamond 16
experience add ${player} 5 levels
tellraw ${player} {"text":"Покупка выполнена","color":"green"}
```

Если выполнение началось и одна из команд завершилась неуспешно, транзакция переходит в `REVIEW_REQUIRED`. Уже выполненные команды автоматически не откатываются и не повторяются.

## 10. ITEM-иконки

Для предметной иконки установите:

```text
поле: icon.type
значение: "ITEM"
```

Затем registry ID:

```text
поле: icon.item
значение: "minecraft:diamond"
```

И количество:

```text
поле: icon.count
значение: 16
```

Сервер проверяет существование предмета. `display_nbt` ограничен по размеру и глубине настройками `server.json`. Он предназначен только для безопасного отображения и не определяет результат покупки.

## 11. TEXTURE-иконки

Для текстурной иконки:

```text
поле: icon.type
значение: "TEXTURE"
```

```text
поле: icon.texture
значение: "example:textures/gui/shop_icon.png"
```

Текстура должна находиться в установленном моде или ресурспаке и быть доступна клиенту. Редактор не загружает PNG с файловой системы сервера и не пересылает изображения по сети.

## 12. Оформление

Оформление наследуется в следующем порядке:

1. Глобальный `style`.
2. `style` категории.
3. `style` товара.

Пустая строка означает отсутствие цветового или текстурного переопределения.

Поля стиля:

| Поле | Тип | Назначение |
|---|---:|---|
| `background_color` | string | Общий фон. |
| `panel_color` | string | Фон панели. |
| `category_list_color` | string | Фон списка категорий. |
| `tab_color` | string | Обычная вкладка. |
| `tab_hover_color` | string | Вкладка под курсором. |
| `tab_active_color` | string | Активная вкладка. |
| `card_color` | string | Обычная карточка. |
| `card_hover_color` | string | Карточка под курсором. |
| `button_color` | string | Кнопка покупки. |
| `border_color` | string | Цвет рамки. |
| `text_color` | string | Основной текст. |
| `price_color` | string | Цена. |
| `balance_color` | string | Баланс. |
| `texture` | string | Устаревшая общая текстура; оставлена для совместимости. |
| `background_texture` | object | Фон всего магазина. |
| `panel_texture` | object | Основная панель. |
| `category_list_texture` | object | Область списка категорий. |
| `tab_texture` | object | Обычная вкладка категории. |
| `tab_hover_texture` | object | Вкладка под курсором. |
| `tab_active_texture` | object | Активная вкладка. |
| `card_texture` | object | Обычная карточка товара. |
| `card_hover_texture` | object | Карточка под курсором. |
| `button_texture` | object | Кнопка покупки. |
| `border_width` | int | Толщина рамки. |
| `padding` | int | Внутренний отступ. |
| `opacity` | int | Прозрачность от 0 до 255. |

Цвета принимаются только в форматах:

```text
#RRGGBB
#AARRGGBB
```

Восьмизначный формат является ARGB, а не RGBA.

Для стиля товара магазин использует только `card_color`, `card_hover_color`, `button_color`, `border_color`, `text_color`, `price_color`, соответствующие текстуры карточки/кнопки, `border_width` и `opacity`. Остальные поля `StyleDefinition` имеют смысл только на уровне категории или глобального стиля и намеренно не показываются в обычном редакторе товара.

Текстурный объект содержит `path`, `mode`, `anchor`, `offset_x`, `offset_y`, `scale`. Режимы: `STRETCH`, `COVER`, `CONTAIN`, `ORIGINAL`, `TILE`. Привязки: `TOP_LEFT`, `TOP`, `TOP_RIGHT`, `LEFT`, `CENTER`, `RIGHT`, `BOTTOM_LEFT`, `BOTTOM`, `BOTTOM_RIGHT`. Масштаб задаётся в процентах от 10 до 400. Пустой `path` отключает слот. У категории работают все слоты; у товара — `card_texture`, `card_hover_texture` и `button_texture`.

Пример изменения всего слота одним значением:

```text
поле: style.background_texture
значение: {"path":"minecraft:textures/block/stone.png","mode":"COVER","anchor":"CENTER","offset_x":0,"offset_y":0,"scale":100}
```

### Глобальный стиль

Чтобы изменить глобальный стиль, заполните поле и значение, затем нажмите `Глоб. стиль`.

Пример:

```text
поле: card_color
значение: "#FF252C35"
```

### Стиль категории

Используйте `Поле кат.` и путь с `style.`:

```text
category_id: resources
поле: style.tab_active_color
значение: "#FF315A7D"
```

### Стиль товара

Используйте `Поле тов.`:

```text
category_id: resources
slot_id: diamonds_16
поле: style.card_hover_color
значение: "#FF405A70"
```

Чтобы убрать переопределение цвета:

```json
""
```

## 14. Проверка черновика

Кнопка `Проверить` просит сервер полностью проверить текущий черновик без сохранения.

Проверяются, среди прочего:

- `schema_version` и revision;
- формат, длина и уникальность ID;
- длина названий и описаний;
- отрицательные цены и выход за пределы long;
- координаты, размеры и порядок;
- цвета и ResourceLocation;
- существование ITEM-иконок;
- размер и глубина display NBT;
- имена прав;
- purchase limit, период и cooldown;
- количество категорий и товаров;
- количество и длина команд;
- допустимые заполнители;
- разбор команд серверным `CommandDispatcher`;
- общий размер конфигурации и сетевые ограничения.

Одна ошибочная операция не изменяет серверный черновик. Клиент получит путь проблемного поля и причину ошибки.

## 15. Сохранение

Кнопка `Сохранить` запускает серверную процедуру:

1. Повторная проверка `editor_session_id`.
2. Повторная проверка административного права.
3. Сравнение `base_revision` с текущей revision магазина.
4. Полная валидация черновика.
5. Создание резервной копии.
6. Увеличение revision на 1.
7. Запись во временный файл рядом с `shop.json`.
8. Перечитывание и повторная проверка временного файла.
9. Атомарная замена `shop.json`.
10. Публикация нового неизменяемого `ShopSnapshot`.
11. Запись действия `EDITOR_SAVE` в SQLite audit.
12. Уведомление клиентов с открытым магазином о новой revision.

После успешного сохранения редактор остаётся открытым. Сервер обновляет черновик из опубликованного snapshot, устанавливает новую `base_revision` и отправляет обновлённый `EditorDto`. Следующее сохранение работает в той же сессии уже относительно новой revision. Сессия удаляется только при отмене, закрытии, отключении игрока или истечении тайм-аута.

## 16. Конфликт base_revision

Если магазин изменился после открытия редактора, сохранение будет отклонено:

```text
Конфигурация изменилась. Закройте редактор и откройте его заново.
```

Автоматическое слияние не выполняется. Это защищает изменения другого администратора и ручной reload от случайной перезаписи.

Правильный порядок действий:

1. Не пытаться повторять старое сохранение.
2. Закрыть редактор.
3. Открыть его снова командой `/ms admin editor_extension`.
4. Повторить нужные изменения на основе новой revision.

## 17. Отмена и закрытие

Кнопка `Отменить` отправляет серверу операцию отмены, удаляет черновик и закрывает экран.

Если есть несохранённые изменения и нажать Esc:

1. Первый Esc показывает предупреждение.
2. Второй Esc в течение четырёх секунд подтверждает отмену.

Черновик также удаляется:

- при выходе администратора с сервера;
- при истечении тайм-аута;
- при открытии новой редакторской сессии;
- после успешного сохранения.

Сервер не полагается только на уведомление клиента о закрытии.

## 18. Ограничение частоты операций

Операции редактора защищены rate limit. При стандартном `server.json` допускается до 20 операций за 5 секунд. Значение вычисляется на основе серверного purchase rate limit, но не может быть меньше 10 операций на окно.

Если операции отправляются слишком быстро, сервер ответит:

```text
Слишком много операций редактора. Повторите позже.
```

Отклонённый по rate limit запрос не изменяет черновик.

## 19. Полный пример создания товара

Ниже приведена последовательность для товара «16 алмазов» в категории `resources`.

### Шаг 1. Создать категорию

```text
category_id: resources
кнопка: + кат.
```

### Шаг 2. Назвать категорию

```text
category_id: resources
поле: title
значение: "Ресурсы"
кнопка: Поле кат.
```

### Шаг 3. Настроить иконку категории

```text
поле: icon.type
значение: "ITEM"
```

```text
поле: icon.item
значение: "minecraft:diamond"
```

Задайте общий размер иконок наследующих товаров:

```text
поле: product_icon_size
значение: 24
```

### Шаг 4. Создать товар

```text
category_id: resources
slot_id: diamonds_16
кнопка: + товар
```

### Шаг 5. Назвать товар

```text
поле: title
значение: "16 алмазов"
кнопка: Поле тов.
```

### Шаг 6. Добавить описание

```text
поле: description
значение: ["Выдаёт 16 алмазов", "Цена: {price} XR"]
```

### Шаг 7. Установить цену

```text
поле: price_xr
значение: 100
```

### Шаг 8. Настроить иконку товара

Новый товар уже имеет `icon.size: 0`, поэтому в этом примере размер 24 берётся из категории. Предмет и количество остаются собственными настройками товара.

```text
поле: icon.item
значение: "minecraft:diamond"
```

```text
поле: icon.count
значение: 16
```

### Шаг 9. Добавить команду

В поле значения введите без JSON-кавычек:

```text
give ${player} minecraft:diamond 16
```

Нажмите `+ команда`.

### Шаг 10. Включить товар

```text
поле: enabled
значение: true
```

### Шаг 11. Проверить и сохранить

1. Нажмите `Проверить`.
2. Исправьте все показанные ошибки.
3. Нажмите `Сохранить`.
4. Откройте обычный магазин через `/ms open` и проверьте карточку.

## 20. Частые ошибки

### «Значение не является JSON»

Причина: строка введена без кавычек или JSON повреждён.

Неверно:

```text
Ресурсы
```

Верно:

```json
"Ресурсы"
```

### «category_id уже существует»

Новый ID категории конфликтует с существующей категорией. Выберите другой ID.

### «slot_id уже существует в категории»

Товар с таким ID уже есть в исходной или целевой категории.

### «Категория не найдена»

Проверьте `category_id`, регистр символов и отсутствие пробелов. ID чувствителен к регистру и допускает только строчные латинские символы.

### «Товар не найден»

Проверьте одновременно `category_id` и `slot_id`. Один и тот же `slot_id` может существовать в другой категории.

### «Нужна хотя бы одна команда»

Включённый товар не может иметь пустой `purchase_commands`. Добавьте команду или временно установите `enabled: false`.

### «Неизвестный заполнитель»

Использован placeholder `${имя}`, которого нет в разрешённом списке. JSON и NBT с обычными `{...}` разрешены и не считаются placeholders.

### «Команда не разбирается CommandDispatcher»

Проверьте синтаксис после подстановки заполнителей. Команда должна существовать на сервере и не должна начинаться с `/`.

### «Конфликт base_revision»

Магазин изменён после открытия редактора. Закройте экран и откройте новую сессию.

### «Сессия редактора недействительна или истекла»

Сессия закрыта, заменена новой или превысила тайм-аут. Повторно выполните `/ms admin editor_extension`.

### «Слишком много операций редактора»

Сработал серверный rate limit. Подождите окончания окна и продолжите.

### Текстура не отображается

Убедитесь, что `ResourceLocation` корректен и ресурс действительно находится в моде или ресурспаке каждого клиента. Файл на диске сервера клиенту недоступен.

## 21. Рекомендации администраторам

- Перед массовыми изменениями выполните `/ms admin backup`.
- Создавайте новые товары выключенными.
- Всегда используйте `Проверить` перед `Сохранить`.
- После сохранения открывайте магазин обычной командой и проверяйте несколько GUI Scale.
- Для опасных команд сначала создайте тестовый выключенный товар на отдельном сервере.
- Не используйте команду, результат которой невозможно однозначно проверить.
- Не рассчитывайте на автоматический откат внешних команд.
- Регулярно просматривайте `/ms admin review list`.
- Не передавайте право редактора обычным игрокам.
