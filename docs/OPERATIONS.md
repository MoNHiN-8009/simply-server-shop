# Эксплуатация, безопасность и проверки

Обычный клиент никогда не получает `purchase_commands`, серверные пути, JSON, SQLite или скрытые товары. При каждом `/ms open` создаётся новая сессия и новый DTO. После закрытия/отключения DTO очищается; серверные сессии дополнительно истекают по тайм-ауту. Старые пакеты игнорируются.

SimpleChannel версионирован. Направление пакета фиксируется при регистрации; C2S проверяет sender, право, UUID сессии, размер строк и тип операции. DTO ограничен по общему размеру и передаётся частями. Одновременно клиент собирает не более четырёх временных передач.

SQLite обслуживает ограниченный single-thread executor, WAL и индексы. Minecraft-команды выполняются только в серверном потоке; файловые сохранения и SQL не выполняются в кадре или цикле рендера. Snapshot не изменяется после публикации, backup-кэш ограничен `max_backups`.

## Проверка сборки

- `gradlew.bat clean test` — модульные и интеграционные тесты SQLite;
- `gradlew.bat build` — компиляция, ресурсы, тесты, reobf и Jar-in-Jar;
- `gradlew.bat runServer` — ручной smoke-test выделенного сервера (после принятия EULA в dev-каталоге).

Тесты покрывают пустой магазин, строгую загрузку и повреждение JSON, schema, дубликаты ID, цену, overflow, недостаток XR, конкурентное списание, повтор активной покупки, команды/placeholders, частичное выполнение, recovery, revision/STORE_UPDATED, права/фильтрацию DTO, сессии, editor save/conflict, backup recovery и адаптивную раскладку для 1280×720, 1920×1080 и 2560×1440 при нескольких GUI Scale.
