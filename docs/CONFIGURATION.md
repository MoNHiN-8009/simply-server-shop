# Серверные JSON

Все три файла находятся только на сервере в `config/xrshop`. После проверки `shop.json` публикуется как неизменяемый снимок; открытия и покупки не читают файл с диска.

## shop.json

Корень содержит `schema_version` (поддерживается `1`), монотонный `revision`, `ui`, `style` и `categories`. Первый запуск создаёт пустой массив категорий.

`ui` задаёт русские тексты магазина, ARGB-цвета, ширину вкладок и карточек, интервалы, поля, сетку, прокрутку/пагинацию, подтверждение, затемнение мира и звуки. `grid_left_padding` задаёт отступ первой колонки товаров от левого края панели и учитывается при расчёте доступной ширины и числа колонок. Цвета принимаются только как `#RRGGBB` или `#AARRGGBB`; второй формат именно ARGB.

`style` задаёт фон, панели, вкладки, hover/active, карточки, кнопку, рамки, текст, цену, баланс, прозрачность и отдельные доступные клиенту текстуры. Порядок разрешения стиля: товар → категория → глобальный стиль → безопасный цвет UI. Пустой цвет означает отсутствие переопределения.

Текстурные слоты глобального стиля и категории: `background_texture`, `panel_texture`, `category_list_texture`, `tab_texture`, `tab_hover_texture`, `tab_active_texture`, `card_texture`, `card_hover_texture`, `button_texture`. У товара поддерживаются `card_texture`, `card_hover_texture` и `button_texture`. Благодаря этому категория может иметь собственный фон, а товар — собственную карточку, не изменяя остальные элементы.

Каждый слот является объектом:

```json
{
  "path": "minecraft:textures/block/stone.png",
  "mode": "COVER",
  "anchor": "CENTER",
  "offset_x": 0,
  "offset_y": 0,
  "scale": 100
}
```

`path` — клиентский `ResourceLocation` PNG, пустая строка отключает слот. `mode`: `STRETCH` (растянуть), `COVER` (заполнить с обрезкой), `CONTAIN` (вписать целиком), `ORIGINAL` (исходный размер) или `TILE` (повторять). `anchor` принимает `TOP_LEFT`, `TOP`, `TOP_RIGHT`, `LEFT`, `CENTER`, `RIGHT`, `BOTTOM_LEFT`, `BOTTOM`, `BOTTOM_RIGHT`. Смещения задаются в пикселях, `scale` — в процентах от 10 до 400. Старое строковое поле `texture` продолжает читаться для совместимости, но новый редактор его не создаёт.

Категория содержит:

- уникальный `category_id` формата `[a-z0-9_.-]+`, отдельный русский `title`, строки `description`;
- `enabled`, `order`, необязательное `permission`;
- `icon`, `style`, `tab_width`, `padding`, общий `product_icon_size`, массив `products`.

Товар содержит:

- уникальный внутри категории `slot_id`; отдельного `product_id` нет;
- `title`, `description`, `enabled`, целочисленный `price_xr` типа long;
- визуальные `order`, `page`, `row`, `column`;
- `permission`, `confirmation_enabled`, `purchase_button_text`;
- `purchase_limit`, `limit_period_seconds`, `cooldown_seconds`;
- сообщения, звук, иконку, стиль и `purchase_commands`.

Иконка `ITEM` принимает registry ID, количество, размер и ограниченный display NBT. Для товарной `icon.size` значение `0` означает наследование `product_icon_size` текущей категории, а значения от 8 до 64 задают собственный размер. `product_icon_size` категории также принимает от 8 до 64 и по умолчанию равен 16. Наследуется только размер: предмет, текстура, количество и NBT всегда остаются настройками самого товара. Итоговый размер разрешается сервером до отправки DTO и безопасно ограничивается фактической высотой карточки. Иконка `TEXTURE` принимает только доступный клиенту `ResourceLocation`; серверные PNG не пересылаются. Иконка никогда не определяет выдаваемый результат.

## purchase_commands

Команды записываются без начального `/`, выполняются серверной консолью последовательно и являются единственным способом выдачи результата. Поддерживаются только `${player}`, `${uuid}`, `${price}`, `${category_id}`, `${slot_id}`, `${transaction_id}`. Синтаксис `{player}` не поддерживается. Обычные фигурные скобки JSON и NBT не считаются placeholders и остаются без изменений. Строки игрока не подставляются. При загрузке/сохранении выполняются проверки длины, количества, placeholders и разбора `CommandDispatcher`; перед списанием проверка повторяется с настоящими значениями.

Пример: `give ${player} minecraft:diamond 16`. Команды `tellraw ${player} {"text":"Готово"}` и `title ${player} title {"text":"ТЕСТ"}` валидны: JSON не конфликтует с placeholders. Если нужна команда другого Forge-мода, она также работает, если зарегистрирована в серверном dispatcher.

## server.json

Задаёт максимальный баланс, тайм-ауты сессий, rate limit, размер ограниченной SQLite-очереди, busy timeout/WAL, число backup и настраиваемые пределы ID, строк, категорий, товаров, команд, NBT, JSON, сетевых частей, координат, cooldown/limit period. В Java дополнительно действуют абсолютные максимумы; небезопасное увеличение файла отклоняется.

## messages.json

Содержит изменяемые русские серверные сообщения. `{balance}`, `{seconds}` и `{transaction_id}` заменяются сервером в соответствующем контексте. Стандартные клиентские строки находятся в `assets/xrshop/lang/ru_ru.json`.

## Безопасное ручное изменение

Скопируйте текущий файл, измените его, увеличьте `revision`, затем выполните `/ms admin validate` и `/ms admin reload`. Невалидный файл не заменяет рабочий snapshot. Внутриигровой редактор предпочтительнее: он сам создаёт backup и увеличивает revision.
