# Удобный редактор XR Shop

Новый редактор открывается командой:

```text
/ms admin editor
```

Требуется одно из прав: главное `minecraftshop.admin` либо точечное `minecraftshop.admin.editor`. Редактор работает с серверным черновиком: клиент не получает доступ к файлу `shop.json`, а каждое изменение повторно проверяется сервером.

Старый технический экран сохранён:

```text
/ms admin editor_extension
```

Ветка `/ms admin` не передаётся в клиентское дерево команд без единого административного права. Главное право `minecraftshop.admin` открывает все административные команды; точечное право, например `minecraftshop.admin.editor`, показывает только разрешённую операцию.

Его отдельное руководство: [расширенный редактор](EDITOR_EXTENSION.md).

## Устройство экрана

- Слева находится дерево категорий и товаров.
- Сверху слева — поле ID и кнопки создания.
- Справа открывается форма выбранной категории, товара, UI или стиля.
- Параметры разделены на понятные вкладки.
- `?` рядом с полем показывает подробную подсказку.
- Любые введённые значения сохраняются в форме при переходе между вкладками и товарами.
- Над нижними кнопками находится отдельная строка состояния с результатами проверки и ошибками.
- Внизу находятся `Проверить`, `Сохранить` и `Отменить`.

У отдельных полей больше нет галочек. Сервер получает все изменённые поля одним проверяемым пакетом только после нажатия общей кнопки `Сохранить`. `Проверить` проверяет тот же пакет без записи на диск. После успешного сохранения редактор остаётся открытым, очищает отметки сохранённых полей и продолжает работу уже с новой revision.

## Создание категории

1. Введите сверху слева уникальный ID, например `resources`.
2. Нажмите `+ Категория`.
3. На вкладке `Основное` задайте название, описание, порядок и право доступа.
4. В разделе `Размеры` задайте `Размер иконок товаров` для наследующих товаров категории.
5. На вкладке `Иконка` выберите ITEM или TEXTURE для значка самой категории.
6. На вкладке `Оформление` при необходимости переопределите общий стиль.

Описание вводится без JSON. Если нужно несколько строк, разделите их `|`:

```text
Материалы для крафта | Обновляются каждый день
```

## Создание товара

1. Выберите слева категорию.
2. Введите сверху слева ID, например `diamonds_16`.
3. Нажмите `+ Товар`.
4. Заполните `Название`, `Описание` и `Цена XR`.
5. Выберите иконку. Оставьте `Размер предмета` равным `0`, чтобы взять размер категории, либо укажите собственный размер.
6. Добавьте хотя бы одну команду.
7. Только после этого переключите `Товар включён` на `Да`.
8. Нажмите общую кнопку `Сохранить` внизу экрана.

Новые товары создаются выключенными, чтобы игроки не увидели незаконченную карточку.

## Выбор предмета

Поле `Предмет` на вкладке иконки не требует ручного ввода registry ID. Нажмите на него:

1. Откроется сетка всех зарегистрированных предметов.
2. Поиск понимает видимое название и registry ID.
3. Наведите курсор для обычной Minecraft-подсказки.
4. Нажмите на иконку, чтобы вернуться в форму.
5. Выбранный предмет останется в форме; завершив настройку, нажмите общую кнопку `Сохранить`.

Поле товара `Размер предмета` принимает `0` для наследования из категории либо собственный размер от 8 до 64 логических пикселей. Настройка категории находится в `Размеры → Размер иконок товаров` и применяется ко всем её товарам со значением `0`. Наследуется только размер — выбранный предмет, текстура, количество и NBT товара не меняются. Заголовок и цена автоматически сдвигаются под иконку. Если высоты карточки недостаточно, отрисовываемый размер безопасно уменьшается, чтобы иконка, текст, цена и кнопка не перекрывались.

## Конструктор give

1. Откройте товар и вкладку `Команды`.
2. Нажмите `Конструктор` справа от командного поля.
3. Выберите `Выдать предмет`.
4. В `Кому` уже вписано `${player}`. Обычно менять его не нужно.
5. Нажмите на поле предмета и выберите его из сетки.
6. Укажите количество. NBT оставьте пустым, если он не нужен.
7. Нажмите `Проверить и вставить`.
8. Сервер проверит команду через `CommandDispatcher` и поместит её в верхнее поле.
9. Нажмите `+ Добавить`.

Пример результата:

```text
give ${player} minecraft:diamond 16
```

## Конструктор effect

1. На вкладке `Команды` раскройте `Конструктор`.
2. Выберите `Наложить эффект`.
3. Оставьте `${player}` или задайте другую Minecraft-цель.
4. Начните писать название эфекта. Снизу появятся registry ID и переведённые названия.
5. Выберите вариант мышью или нажмите `Tab` для первой подсказки.
6. Укажите длительность в секундах. `30` означает 30 секунд.
7. Укажите понятный человеку уровень: `1` означает первый уровень, `2` — второй. Конструктор сам переведёт его в нужный Minecraft amplifier.
8. При желании скройте частицы.
9. Нажмите `Проверить и вставить`, а затем `+ Добавить`.

Пример:

```text
effect give ${player} minecraft:absorption 30
```

Список автодополнения имеет непрозрачный фон и отдельную полосу прокрутки. Его можно прокручивать колёсиком либо перетаскиванием ползунка; он рисуется поверх формы и не смешивается с текстом полей и кнопок.

## Ручные команды

Конструктор не ограничивает мод только `give` и `effect`. Любую другую серверную команду можно ввести верху без символа `/`.

Доступны placeholders:

- `${player}`;
- `${uuid}`;
- `${price}`;
- `${category_id}`;
- `${slot_id}`;
- `${transaction_id}`.

Старый вид `{player}` не поддерживается. Только последовательность `${имя}` является placeholder. Поэтому JSON-компоненты `{"text":"ТЕСТ","color":"green"}` и NBT `{CustomModelData:1}` можно вставлять в команду без экранирования.

Относительные координаты `~ ~ ~` считаются от серверного источника команды. Чтобы создать частицы возле покупателя, используйте, например: `execute at ${player} run particle minecraft:flame ~ ~1 ~ 0 0 0 0 1 force`.

Команды выполняются от имени сервера по порядку. Каждая добавляемая или заменяемая команда проверяется сервом.

## Остальные вкладки товара

- `Размещение`: порядок, страница, строка и столбец.
- `Ограничения`: лимит покупок, период, cooldown и индивидуальные сообщения.
- `Иконка`: ITEM/TEXTURE, предмет, количество, размер, display NBT и текстура.
- `Звук`: registry ID звука, громкость и высота.
- `Оформление`: только параметры, применимые к товару — карточка, наведение, кнопка, рамка, текст, цена, отдельные текстуры, толщина рамки и прозрачность.

## Выбор цветов

Нажмите на любое поле цвета. Откроется единый подборщик с тремя режимами:

- `Круг` — оттенок выбирается по окружности, насыщенность — расстоянием от центра; рядом находятся яркость и прозрачность.
- `Квадрат HSV` — насыщенность и яркость выбираются в квадрате, оттенок и прозрачность — вертикальными полосами.
- `Ползунки` — точная настройка каналов A, R, G и B от 0 до 255.

Сверху всегда доступно точное значение `#AARRGGBB`. Например, `#FFFFAA00` — полностью непрозрачный оранжевый, а `#8055AAFF` — полупрозрачный голубой. Кнопка `Убрать`, расположенная слева от `Применить`, очищает локальное значение: тогда используется цвет вышестоящего уровня или безопасный цвет интерфейса. `Применить` возвращает выбранное значение в форму; окончательная запись выполняется только общей кнопкой `Сохранить`.

## Выбор и прослушивание звуков

Нажмите на поле звука в товаре или в разделе `Интерфейс → Звуки`:

1. Найдите звук по части registry ID, например `levelup`, `click` или `minecraft:entity.player`.
2. Нажмите строку результата — звук сразу проиграется.
3. Отрегулируйте `Громкость` и `Высоту` и нажмите `▶ Прослушать`, чтобы сравнить результат.
4. `Без звука` очищает настройку.
5. Нажмите `Применить`, затем общую кнопку `Сохранить`.

Громкость `1.00` — обычная, `0.50` — вдвое тише. Высота `1.00` — исходная; меньшее значение делает звук ниже, большее — выше.

Справа у списка находится отдельная полоса прокрутки. Колёсико прокручивает список, а ползунок можно перетаскивать мышью; полоса занимает собственную область и не закрывает названия звуков.

## Выбор текстур и путей

Поля текстур открывают внутриигровой проводник ресурсов. Он показывает PNG из Minecraft, установленных модов и активных ресурспаков, поэтому одинаково работает в Windows, Linux и macOS и не зависит от системного файлового диалога.

Можно искать по `namespace:path`, выбрать строку и увидеть точный предпросмотр. Справа от списка находится отдельная полоса прокрутки: строки заканчиваются до неё и не перекрываются ползунком. Кнопка `Пусто — без текстуры` означает, что мод ничего не рисует: отсутствующая текстура больше не заменяется розово-чёрной заглушкой.

Для каждой текстуры отдельно настраиваются:

- режим `Растянуть`, `Обрезать`, `Вписать`, `Исходный размер` или `Плитка`;
- привязка к одному из девяти положений, например к центру или правому верхнему углу;
- смещение по X и Y;
- масштаб от 10 до 400 процентов.

`Обрезать` заполняет всю область с сохранением пропорций и обрезает выступающие края. `Вписать` показывает изображение целиком, оставляя свободное место. `Исходный размер` использует реальные размеры PNG с учётом масштаба. `Плитка` повторяет изображение. Предпросмотр использует тот же алгоритм, что и магазин.

В глобальном стиле и стиле категории доступны отдельные слоты для фона магазина, панели, списка категорий, обычной/наведённой/активной вкладки, обычной/наведённой карточки и кнопки. Поэтому фон одной категории можно изменить, не затрагивая остальные. В стиле товара доступны отдельные текстуры обычной/наведённой карточки и кнопки. Разрешение идёт от товара к категории и затем к глобальному стилю.

## Общий UI и стиль

Кнопки `Интерфейс` и `Общий стиль` расположены слева над деревом.

В `Интерфейсе` доступны:

- все тексты;
- цвета;
- размеры вкладок, карточек, отступов и сетки;
- страницы, прокрутка, подтверждение и затемнение мира;
- звуки открытия, успеха и ошибки.

В `Общем стиле` доступны все цвета панелей, вкладок, карточек, кнопок, текста, цены и баланса, отдельные текстурные слоты, рамка, отступ и opacity.

`Интерфейс → Размеры → Отступ товаров слева` задаёт свободное место между началом панели товаров и первой карточкой. Значение участвует в расчёте рабочей ширины сетки и количества столбцов; стандартное значение — 8 пикселей.

## Копирование, перенос и удаление

- Выберите категорию/товар, введите новый ID в верхнее поле справа и нажмите `Копировать`.
- Для переноса товара введите в это же поле ID целевой категории и нажмите `Перенести`.
- Удаление требует двух нажатий в течение трёх секунд.
- Копии товаров создаются выключенными.

## Проверка и сохранение

1. Нажмите `Проверить`.
2. Исправьте все указанные сервером поля.
3. Нажмите `Сохранить`.

Сервер применит все изменённые поля к отдельной копии черновика, полностью проверит итог, сравнит `base_revision`, создаст резервную копию и атомарно заменит `shop.json`. Редактор не закрывается: сервер обновляет его черновик и `base_revision`, после чего можно сразу продолжать настройку. Если хотя бы одно поле неверно, не будет сохранено ни одно из них, а введённые значения останутся на экране для исправления.

Если магазин изменился после открытия сессии, сервер отклонит сохранение с конфликтом revision. Откройте новую сессию и повторите изменения.

## Безопасное закрытие

- `Отменить` отбрасывает серверный черновик.
- При наличии изменений первое нажатие `Esc` показывает предупреждение.
- Второе нажатие `Esc` в течение четырёх секунд отменяет сессию.
- Переход в меню выбора предмета не закрывает сессию и не теряет введённые значения.
